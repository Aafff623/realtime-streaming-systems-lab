# Archive policy

## 保留

- 上游 ZIP: `assets/backup/`
- 上游 Word/PDF: `upstream-docs/`
- 可迁移知识: `docs/knowledge/`
- 已批准决策: `docs/adr/`
- 已完成交付: `docs/output/handoff/<theme>/`
- 提交摘要: `docs/commit-history/`

## 原则

- 原始上游资产只读，保持文件内容不变。
- 结构化知识必须标注来源，并指向代码事实源。
- 过期文档不直接删除，先标明替代文档和过期原因。
- 不归档密钥、生产数据和个人身份信息。
