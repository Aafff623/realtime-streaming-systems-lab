# Issue tracker

## 选择

当前使用本地 `.scratch/issues/` Markdown，不连接外部 Issue Tracker。

## 文件命名

```text
.scratch/issues/YYYYMMDD-short-title.md
```

## 必填字段

- 标题
- Category: `bug` 或 `enhancement`
- State
- 背景或问题描述
- 证据与关键路径
- 验收条件
- 关联 report、PRD、handoff

## 生命周期

本地 Issue 只负责工作入口。形成稳定结论后，把知识写入 `docs/`，不要把 `.scratch/` 当长期事实源。
