## Problem

上游代码、静态前端和课程资料彼此分离，README 只给出技术名词，没有说明服务边界、真实链路、运行前置条件和代码风险。直接阅读模块会花大量时间，还容易把课程意图误认为当前实现。

## Objective

产出一份可复核的深度项目分析，让读者能够：

- 理解系统架构与服务依赖。
- 追踪登录、直播、送礼、支付、IM、红包和带货链路。
- 看懂数据模型、缓存、分库分表和消息一致性设计。
- 识别可运行性、测试、安全和技术债风险。
- 获得按价值与依赖排序的学习路线。

## Audience

学习直播微服务架构的开发者，以及后续接手分析、修复或现代化改造的 Agent。

## Scope

### Included

- 后端模块、配置与数据脚本。
- Web 页面、API 和 WebSocket 协议。
- 上游课程资料与源码之间的对应关系。
- 架构、业务、数据、工程质量、安全和运行风险。

### Excluded

- 直接修复业务 Bug。
- 迁移 Vue 3 或升级后端依赖。
- 搭建完整中间件环境。
- 生产部署与容量承诺。

## Deliverables

1. 系统上下文与服务拓扑。
2. 核心业务链路说明。
3. 数据库、缓存、MQ 和一致性分析。
4. 前后端协议对照。
5. 可运行性与配置差异清单。
6. 测试、安全与技术债评估。
7. 分阶段学习和改造建议。

## Acceptance criteria

- 每个关键判断都能追溯到文件或配置。
- 明确区分实现事实、课程意图和建议。
- 至少覆盖登录、送礼和 IM 三条端到端链路。
- 风险按影响和验证成本排序。
- 不修改业务代码。
